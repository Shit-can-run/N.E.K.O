# Плагин Mijia · Краткое руководство по использованию (v1.1)

> **Ключевая подсказка**: данный плагин построен на **управлении естественным языком**. В большинстве случаев вам не нужно искать ID устройства или технические параметры — достаточно произнести команду обычными словами.

---

## 1. Шпаргалка по основным командам

Просто скажите AI Agent одну из приведённых ниже команд, и плагин автоматически разберёт её и выполнит:

| Тип действия | Пример команды (можно скопировать) | Пояснение |
| :--- | :--- | :--- |
| **💡 Включение/выключение устройства** | `включи свет в спальне` <br> `выключи розетку в гостиной` | Поддерживаются глаголы «включи / выключи / вкл / выкл» |
| **🏠 Управление по зонам** | `включи свет в гостиной` <br> `выключи кондиционер в главной спальне` | **(Акцент новой версии)** Поддерживается «название комнаты + название устройства» |
| **🌡️ Регулировка температуры** | `установи кондиционер на 26 градусов` <br> `выстави кондиционер на 24 градуса` | Числа и единицы измерения распознаются автоматически |
| **☀️ Регулировка яркости** | `установи свет на 50%` <br> `яркость настольной лампы 30` | Поддерживаются проценты и целые числа |
| **🔄 Смена режима** | `переключи кондиционер в режим охлаждения` <br> `переведи вентилятор в авто` | Поддерживаются распространённые названия режимов |
| **🎬 Запуск сценария** | `запусти сценарий «дома»` <br> `активируй сценарий «ушёл из дома»` | Сценарии необходимо заранее настроить в приложении Mijia App |

## 2. Продвинутый приём: решаем проблему «слишком много устройств, не нашли нужное»

Если у вас дома несколько устройств одного типа (например, несколько «светильников»), команда `включи свет` может вызвать запрос на уточнение.

### 1. Пример запроса на уточнение
Если команда неоднозначна, плагин вернёт сообщение, похожее на следующее, — **добавьте название комнаты** согласно подсказке:

> Найдено 3 устройства, соответствующих «свет»:
> 1. 🟢 Гостиная — потолочный светильник
> 2. 🔴 Спальня — потолочный светильник
> Уточните, пожалуйста, по схеме «название комнаты + название устройства», например «свет в спальне»

### 2. Рекомендуемый формат точной команды
Чтобы избежать путаницы, рекомендуем выработать привычку использовать **`[название комнаты] + [название устройства]`**:

*   ❌ Расплывчатая команда: `включи свет`
*   ✅ Точная команда: `включи свет в спальне`
*   ✅ Точная команда: `включи потолочный светильник в гостиной`

## 3. Часто используемые точки входа API (для разработчиков и продвинутых пользователей)

Если вам нужно вызвать интерфейс плагина напрямую, основные точки входа описаны ниже:

| ID точки входа | Функция | Пример параметров |
| :--- | :--- | :--- |
| `smart_control` | **Единое управление естественным языком (рекомендуется)** | `{ "command": "включи свет в спальне" }` |
| `find_device_by_name` | Поиск устройства | `{ "name": "свет в спальне" }` |
| `query_device_state` | Запрос состояния устройства | `{ "name": "кондиционер" }` |
| `execute_scene` | Запуск сценария | `{ "scene_id": "12345" }` |

## 4. Часто задаваемые вопросы (FAQ)

**Q: Почему появляется сообщение «не выполнен вход»?**
A: Сначала откройте плагин Mijia на панели плагинов NEKO, нажмите «Войти по QR-коду» и завершите авторизацию.

**Q: Почему устройство офлайн, а плагин показывает, что оно онлайн?**
A: Плагин показывает облачное состояние, в котором возможны задержки. Проверьте, действительно ли устройство подключено к питанию и к приложению Mijia App.

**Q: Какие устройства поддерживаются?**
A: Теоретически поддерживаются все устройства IoT-платформы Mijia. Любое устройство, которым можно управлять в приложении Mijia App, поддерживается и данным плагином.
